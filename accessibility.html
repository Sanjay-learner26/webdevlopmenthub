Accessibility Basics (SEO Related)

Accessibility improves website usability for all users. Using proper heading structure and alt text improves SEO. Search engines prefer well-structured and accessible websites.


Example 
<img src="logo.png" alt="Company Logo">

<h1>Main Title</h1>
<h2>Sub Heading</h2>

Accessibility (A11Y)

Accessibility (A11Y) means making websites usable for everyone, including people with disabilities. It improves user experience and also helps SEO. Semantic HTML and proper attributes improve accessibility.


Feature: alt Text

The alt attribute describes images for screen readers. It helps visually impaired users understand image content. It also appears if the image fails to load.

Code Example 
<img src="student.jpg" alt="Student learning HTML on laptop">

Attribute: aria-label

The aria-label attribute provides extra information for assistive technologies. It is used when visible text is not enough. It improves screen reader support.

Code Example 
<button aria-label="Close Menu">X</button>

Semantic Structure

Using semantic tags like <header>, <nav>, and <main> improves structure. It helps screen readers understand page layout. Proper structure makes websites more accessible.

Code Example 
<header>
  <h1>My Website</h1>
</header>

<main>
  <p>Welcome to my page</p>
</main>

Proper Label Usage

Form inputs should always have <label> elements. Labels improve accessibility and usability. Clicking the label focuses the input field.

Code Example 
<label for="email">Email:</label>
<input type="email" id="email">
