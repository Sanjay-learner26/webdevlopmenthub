FUNCTIONS

Functions are essential, reusable blocks of code in JavaScript that perform specific tasks, enable modularity, and help avoid code repetition. They are executed when they are called or "invoked".

What is a Function?

A function is a reusable block of code designed to perform a specific task. Instead of repeating code, we create a function.

Function Declaration
Example: 
        function greet(){
  console.log("Hello Sanjay");
}

greet();

Parameters & Arguments

In JavaScript, parameters are the named variables in a function's definition, while arguments are the actual values passed to the function when it is called.

Content:
  • Parameter → variable in function definition
  • Argument → value passed when calling
Example: 
    function greet(name){
  console.log("Hello " + name);
}

greet("Sanjay");

Return Statement

The return statement in JavaScript is used to stop the execution of a function and send a value back to where the function was called. Every function in JavaScript returns a value; if no explicit return statement is used, the function automatically returns undefined.

Returns value from function.
Example: 
        function add(a, b){
  return a + b;
}

let result = add(5, 3);
console.log(result);

Function Expression

A function expression in JavaScript defines a function as part of an expression, often by assigning it to a variable. Unlike a function declaration, it can be anonymous (without a name) and is not hoisted to the top of its scope, meaning it can only be called after it is defined.

Example: 
        const greet = function(){
  console.log("Hello");
};

Arrow Function (ES6)

Arrow functions are a concise alternative to traditional function expressions in JavaScript, introduced in ES6 (ECMAScript 2015). They use the => ("fat arrow") syntax and provide several benefits, primarily a shorter syntax and the lexical binding of the this keyword.

Example: 
        const add = (a, b) => {
  return a + b;
};

Default Parameters

In JavaScript, default parameters allow function parameters to be initialized with default values if no value or undefined is passed during the function call. This feature was introduced in ECMAScript 2015 (ES6) and provides a cleaner syntax compared to older methods of checking for missing arguments within the function body.

Example: 
        function greet(name = "Guest"){
  console.log("Hello " + name);
}

Callback Function

A callback function in JavaScript is a function passed as an argument to another function, which is then executed (or "called back") at a later point in time by the receiving function. This mechanism is fundamental to JavaScript's ability to handle operations like event processing and asynchronous tasks without blocking the main program's execution.

Function passed as argument
Example: 
        function greet(name, callback){
  console.log("Hello " + name);
  callback();
}

function sayBye(){
  console.log("Bye!");
}

greet("Sanjay", sayBye);

Scope

Scope in JavaScript is the context that determines the accessibility and visibility of variables, functions, and objects in different parts of your code. Understanding scope is crucial for writing clean, efficient, and bug-free code by preventing naming conflicts and managing variable lifetimes.

Local Scope

Variable inside function.

Global Scope

Variable outside function.

Example: 
        let globalVar = "Hello";

function test(){
  let localVar = "Hi";
}
